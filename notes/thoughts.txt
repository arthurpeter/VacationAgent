I want to design and implement the FINAL STAGE of my AI itinerary planner app — the actual schedule generation + interactive planning experience after the user finishes selecting attractions, pace, and mobility preferences.

Current Context

* The app already has:

  * attraction selection
  * AI recommendations for mobility and pace
  * transport configuration
  * pace selection
  * ability to fetch transport/rental details
* The backend stores everything in a centralized itinerary state object.

Relevant state shape:

```python
class ItineraryState(TypedDict):
    messages: Annotated[List[Union[dict, str]], add_messages]
    user_id: str
    session_id: int
    stage: int = 0

    pois: List[dict[str, Any]] = []
    unresolved_attractions: List[dict[str, Any]] = []
    resolved_attractions: Annotated[List[dict[str, Any]], operator.add] = []

    mobility_config: Optional[dict] = None
    mobility_recommendation: Optional[MobilityRecommendationSchema] = None

    pace_recommendation: Optional[PaceRecommendationSchema] = None
    pace: Literal["Relaxed", "Moderate", "Fast-Paced"] = "Moderate"
```

What I Want This Final Stage To Become
I want this step to feel like an intelligent interactive trip planner — not just a static generated itinerary.

Main Idea
The flow should work like this:

1. Generate an INITIAL rough itinerary

* Group attractions by day
* Use heuristics + approximate travel times
* Respect:

  * user pace
  * opening hours
  * attraction durations
  * geographic clustering
  * transport type
* This first pass should be fast and good enough for interaction.

2. Interactive itinerary editing
   User can:

* move attractions between days
* reorder attractions
* lock attractions
* remove/add attractions
* regenerate a single day
* regenerate the entire trip

3. Interactive map experience
   This is one of the key differentiators I want.

I want:

* a map per day
* markers for attractions
* visible route lines between attractions
* timeline/order visualization
* potentially animated route progression

Potential stack:

* React
* Leaflet
* Routing engine / APIs

The map should:

* visually show the route
* make the itinerary “feel real”
* immediately show if a day is geographically bad
* update dynamically when attractions move

4. Transit refinement phase
   After the rough itinerary exists:

* fetch REAL transport data
* refine actual travel durations
* refine timing windows
* detect impossible schedules

This includes:

* public transport
* walking
* driving
* potentially mixed transport

Important:
I need REAL public transport times eventually.
Approximate heuristics are acceptable for initial generation, but refinement should become accurate.

5. Smart schedule validation
   The system should detect:

* impossible timing
* attractions too far apart
* overloaded days
* closed attractions
* unrealistic transit windows

Then surface warnings like:

* “This museum closes before arrival.”
* “Transit between these attractions is unusually long.”
* “This day exceeds your selected pace.”

6. UX Goals
   I want this stage to feel:

* premium
* interactive
* intelligent
* visual
* customizable

NOT:

* a plain list of attractions

I want users to feel:
“this planner actually understands my trip.”

7. Final Packaging
   After the user accepts the itinerary:

* generate a polished trip package
* downloadable/shareable
* include:

  * daily schedules
  * maps
  * transport info
  * official links
  * notes/warnings
  * summaries
